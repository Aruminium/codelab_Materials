Node.jsとは

Javascript実行環境です。

ブラウザでしか使えなかったjavascriptだが、Node.jsを用いるとOSの機能が使える。つまりJavascriptの拡張版です

例えばZoomやブラウザ版のDiscordもnode.jsを元に動いているぞ

サーバーサイドとは

Webサイトの裏側の処理、Webページに必要な情報をデータベースから持ってきたり、

Webページを生成するということを行います

フロントエンドとは

よくフロントエンドとサーバーサイドは曖昧な方がいます。筆者もその一人でした。

フロントエンドとはWebサイトの表側の処理を担当します。

「CSSが...」とか「ここはh1タグで...」や「動的なサイトにしよう」とかはフロントエンドです。

またフロントエンドではjavascriptがよく使われます。

といっても実際は標準のjavascriptで書くのではなくVue.jsとかのフレームワーク、Reactなどのライブラリを使ってフロントエンドをつくっていきます

インストール

Windows

  1. node.js←をクリック、そしてLTS(長期サポート版)をダウンロード

  1. ダウンロードファイルを実行してインストーラー起動

基本的にNextを押すだけでよい

チェックを外してNextを押す。「色々とインストールしますがよいですか？」と聞いている。後に必要になったら個別でインストールすれば良い。

mac

windows版と同じ手順でインストールしても良いが、ここではhomebrewを用いた手順を解説する。

Homebrewインストール

  1. Homebrewをクリックする

クリックするとこのページに遷移されるそこで、

の右のアイコンをクリックしてコピーする。

  1. terminalを開いてペーストしてエンターキーでインストールされます

Nodebrewインストール

nodebrewを利用するとnodejsのバージョン管理が便利になります。

以後terminalでの操作の場合は"$"をつけます。($をつけたまま入力しないこと)

$ brew install nodebrew

でインストールされます

$ vi ~/.bash_profile

.bash_profileとかはお使いのターミナルによりますが、要はPathを通します

export PATH=$HOME/.nodebrew/current/bin:$PATH

※bashの場合です。

ターミナルを再起動してから

$ nodebrew install-binary stable

で安定版(LTS)がインストールされます

  1. プロジェクトに招待する

プロジェクトに招待すると、pushやmergeができます。チーム開発するときは招待してあげましょう。参加者(コラボレーター)は自分のgithubのidを教えましょう

この場合下のAruminiumを教えましょう

レポジトリ作成者はそのレポジトリのSettingsをクリックしてCollaboratorsタブから招待しましょう。

  1. git clone

コラボレーターは自分の環境（ローカル環境）にソースコードを持ってくる必要があります。そこで活躍するのがgit cloneです。

任意の場所で

$ git clone リポジトリのurl

とするとローカル環境にリポジトリ内容が来ます。

$ cd リポジトリ名

で準備は完了です。

  1. npm install

$ npm install

とすると必要なパッケージがインストールされます。

  1. README.mdを編集しよう

手始めにREADME.mdを編集し、add,commit, pushすることで一連の流れを確認する。

編集する前に必ずブランチを切ろう。ブランチを切ることでやり直したりできる。

ここでは

$ git checkout -b discordの名前

でブランチを切ろう。

ブランチを切ったらREADME.mdの末尾に

test dicordの名前

を追加しよう。

  1. Pushまで

add -> commit -> push の流れで進む

$ git add README.md
$ git commit -m "やったことを書く"
$ git push origin 先ほど作成したブランチ名

  1. PRを出す

githubのレポジトリを見るとPRがだせるようになっているはずだ。

PRテンプレートが設定されていればそれ通りに入力してPRを出す。

git まとめ

  1. git clone
  2. npm install
  3. git checkout -b
  4. 編集
  5. git add 編集したファイル
  6. git commit "やったこと"
  7. git push origin 作成したブランチ名

またmergeされたファイルの差分を取り組むコマンドgit pullを絶対にわすれないでほしい。

忘れるとエラーやコンフリクトの原因になる。
